> **📌 重要声明 / Important Disclaimer**

> Nonull 是一个**内部使用的 ADAS 工程开发助手**，**不是**经过 ISO 26262 / ASIL-D 认证的车规级安全产品。本项目中的"安全层"是**建议性（advisory）**的——它参考 ISO 26262 / MISRA / ASPICE 等标准的模式与术语进行风险提示和检查建议，但**并不实现** ASIL-D 要求的抗干扰、MC/DC 覆盖、形式化验证、SEooC 等条款。**请勿用于任何量产部署、安全关键决策，或替代经过认证的安全机制。**

## Nonull 是什么？

Nonull 是专门为**智能驾驶（自动驾驶）**行业设计的 AI 智能体。

简单说：**它是一个帮你做代码审查、安全分析、测试设计、场景生成的 AI 工程师。**

- 你用自然语言告诉它任务
- 它自己思考怎么做
- 它调用各种技能去完成

## 和别的 AI 有什么区别？

Nonull 有**两个其他 AI 没有的东西**：

### 1. 🧠 记忆系统
它不会"聊完就忘"。它有四种记忆：
- **工作记忆** — 记得当前任务在干啥
- **情景记忆** — 记得之前做过的类似任务和结果
- **知识记忆** — 记得所有智驾领域知识（法规、标准、算法）
- **技能记忆** — 记得怎么做各种操作

### 2. 🛡️ 安全守护
所有操作都要过五关安全检查，🔍 **安全建议模板内置（基于 ISO 26262 / MISRA 模式，不是认证）**。这是一个建议性（advisory）安全层，不构成车规级安全机制。

> **⚠️ 关于自我进化 / 自我意识模块**：原"自我进化"功能已迁移到 `experimental/` 目录，是**实验性的**，未接入主循环。
> 不推荐将其用于任何安全关键路径。详见 [`experimental/README.md`](../experimental/README.md)。

## 快速上手（2 分钟）

### 1. 安装

```bash
pip install -e .
# 或
pip install -r requirements.txt
```

### 2. 启动

```bash
python -m nonull
# 或
nonull
```

看到 `>>>` 提示符后，就可以开始用了。

### 3. 试试这些

```
>>> 帮我审查这段 C++ 代码有没有安全问题
>>> 对 AEB 系统做 HARA 分析
>>> 列出你有哪些技能
>>> 查看系统状态
```

### 4. Python 代码里用

```python
from core import Nonull

agent = Nonull()

# 同步方式
result = agent.run_sync("分析这段感知代码的性能问题")
print(result["output"])

# 异步方式（推荐）
result = await agent.run("生成 L3 变道场景的测试用例")
```
